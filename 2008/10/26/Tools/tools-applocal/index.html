实用工具介绍 AppLocale

2008-10-26
2020-04-13
工具

【转码】微软AppLocale，SoraApp，W2kXpCjk，还是pAppLocale？

转自http://ent9.com/thread-186760-1-1.html

SoraApp(全名: Sumisora Application Database for 真神)
pAppLocale(全名：piaip AppLocale，有的网页介绍直接以”最強的程式亂碼解決”称呼之)

就功能上来说，Microsoft AppLocale(以下以微软原版简称), SoraApp v1.5, pAppLocale可视为是完全一样的软件。

SoraApp的出现源自这里提到的微软原版(SoraApp v1.5才可脱离Microsoft Applocale单独使用)，主要是去除微软原版的乱码后遗症，而且不会出现微软原版临时措施消息框，并在右键菜单加入”以日文环境运行””以繁体中文环境运行”(但卸载后不会删除，仍会残留在右键菜单，须另用工具或自行手动在注册表内删除)

pAppLocale的出现同样源自这里提到的微软原版，主要也是去除微软原版的乱码后遗症，而且不会出现微软原版临时措施消息框。pAppLocale本身虽不具备添加右键菜单的功能，不过配合辅助配件(参照下方附注)的使用可自由添加功能项于右键菜单或右键”发送到”内。

另外相关软件常见的还有W2kXpCjk(非源自这里提到的微软原版)，虽也可模拟语系运行游戏，但使用上相对而言不是很方便。不过这软件有自己的特定功能，有兴趣的人自行尝试。

微软原版 SoraApp 1.5 pAppLocale pAppLocale+辅助配件
无微软原版乱码后遗症 Ｘ ○ ○ ○
去除临时措施消息框 Ｘ ○ ○ ○
开始菜单 ○ Ｘ ○ ○
右键菜单功能项添加 Ｘ ○ Ｘ ○
右键菜单功能项删除 Ｘ Ｘ Ｘ ○
右键发送到 Ｘ Ｘ Ｘ ○

看起来真的是不推荐使用微软原版，不过精神上还是要强力感谢微软曾推出此产品(不然可能不会有SoraApp及pAppLocale)；也要感激SoraApp的出现，帮助部分玩家解决(免手动)乱码后遗症的困扰，并去除微软原版临时措施消息框。

总结：就目前而言，pAppLocale搭配辅助配件可以彻底取代SoraApp 1.5以及微软原版。

pAppLocale(Microsoft AppLocale 修改版)
◎出处 http://www.csie.ntu.edu.tw/~piaip/
◎下载 http://www.csie.ntu.edu.tw/~piaip/papploc.msi

作者并没有明显介绍此修改版，只在其2004年10月6日的日志提了一小段话

这里提到的【裝了後 MSI Installer 都會用 上次設定的語系跑】，指的就是一般提到的”乱码后遗症”。

这里直接举个例：安装AppLocale前，运行OWC10.MSI(OfficeXP光盘里)出来的画面

安装AppLocale后，以AppLocale指定”日本語”运行战国史，然后关闭，关闭后运行OWC10.MSI出来的画面

原因：此例文字部分之所以变成乱码，是因为被当成Shift-JIS码来解释。为何会被当成 Shift-JIS码呢？是因为AppLocale以”日本語”运行战国史后，在C:\WINDOWS\AppPatch\产生的AppLoc.tmp档记录了”语系”，而MSI档在运行时会参考到该tmp档所带的信息。

解决之道：将AppLoc.tmp删除，重建一个档名为AppLoc.tmp的新档(此时档案大小为0字节)，并将档案属性设置为”只读”，这样以后运行MSI档就不再会有出现乱码的可能。

为何介绍此修改版呢？
因为此修改版不会有安装微软原版所产生的上述乱码后遗症问题，不用手动去做上述动作。

唯一要注意的是：如要卸载，请至”控制面板”->”添加或删除程序”找”piaip AppLocale”(微软原版的是”Microsoft AppLocale”)卸载，然后找”Microsoft Windows Application Compatibility Database”(微软原版此处相同)卸载。

看到这你可能会说那个AppLoc.tmp的问题已经处理过了，不需要去卸载原版重装此版吧？！
如果这个理由不足以说服你，请继续看下去….

用过原版AppLocale的应该对此画面不陌生吧？！

每次运行都会出现此窗口是不是看到、按到有点烦？！

改装修改版就不会有此问题了(也就是修改版作者提到的最后一句话)

改用修改版后就没有烦恼了吗？
以下提供另两种方式运行pAppLocale

一、右键菜单(修改版本身不具备此功能，附件accessory_ch.rar里有提供此功能的reg档)
有不少软件将一些常用功能放到右键菜单方便使用者运行，安装AppLocale后并没有这个功能或开启的选项。
不过已经有人做好reg档让大家可以这么使用了。
◎参见 http://toget.pchome.com.tw/intro/utility_other/23869.html
该出处只提供”日本語執行””簡體中文執行”两种功能的reg档，
为方便使用简体操作系统的使用者，我已加入”繁体中文执行”功能的reg档并增加从右键菜单移除的reg档。

如何添加：想在右键菜单加入”日本語”，直接双击压缩档内的”新增右键_日本语.reg”确认即可。
如何删除：想去掉(觉得右键菜单太杂或其他原因)，就双击压缩档内的”移除右键_日本语.reg”确认即可。

二、右键”发送到”(修改版本身不具备此功能，附件accessory_ch.rar里有提供此功能的wsf档)
如果觉得右键菜单太杂或不想放在那里另一个选择就是右键”发送到”
同样的已经有善心人士做好了可让大家直接取用
◎参见 http://www.hokoy.com/Works/Stuff/MsAppLocale.aspx
此老兄将网站架设在自己的机器上，所以可能连不上，想看该网站介绍的可以用google以”消除程式亂碼的程式”查询，出现的第一个就是(贴此篇时是第一个,不然就看一下网址对不对)点击”网页快照”观看。

如何添加：想在”发送到”加入”中文(繁體)”，就将”AppLocale中文(繁體).wsf”解压缩放到”发送到”所在的文件夹即可。
如何移除：开启”发送到”文件夹，将AppLocale中文(繁體).wsf档案删除即可。

快速开启”发送到”文件夹的方法：Win键+R[开始菜单上面的”运行”]，然后键入”sendto”，接着按”Enter”(回车)
注：Win键就是键盘左下方Ctrl键和Alt键之间带有视窗图标的那个按键。

如果觉得在”发送到”显示的名称不喜欢可以直接改档名，不过要记得.wsf一定要留着才有作用哦！

总结：因此使用AppLocale的方式就可以视情况灵活运用
※开始菜单(可将开始菜单里面做好的快捷方式复制出或搬移到集中管理的文件夹)
※右键菜单(exe执行档或exe执行档的快捷方式按右键)
※右键”发送到”(exe执行档或exe执行档的快捷方式按右键)

附件”accessory_ch.rar”包含”右键菜单”和”右键发送到”所需的档案(显示中文)效果图

附件”accessory_en.rar”包含”右键菜单”和”右键发送到”所需的档案(显示英文)效果图

至于pAppLocale(Microsoft AppLocale 修改版)请从作者网站下载(约1.4MB)或用搜寻引擎查找看看
◎官方下载点 http://www.csie.ntu.edu.tw/~piaip/papploc.msi
